Git Cheat Sheet

Für diejenigen, die die auch die Geburtstage der Familie vergessen :D

<-- Klick dich durch!

SETUP

Konfigurieren von Benutzerinformationen, die in allen lokalen Repositories verwendet werden

git config --global user.name “[Vorname Nachname]”

einen Namen festzulegen, der bei der Überprüfung des Versionsverlaufs für die Anerkennung identifizierbar ist

git config --global user.email “[gültige Email]”

eine E-Mail-Adresse festlegen, die mit jeder Verlaufsmarkierung verknüpft werden soll

git config --global color.ui auto

automatische Einfärbung der Befehlszeile für Git zur leichteren Überprüfung einstellen

git init

ein vorhandenes Verzeichnis als Git-Repository initialisieren

git clone [url]

ein gesamtes Repository von einem gehosteten Standort über eine URL abrufen

LOKALE ÄNDERUNGEN

Veränderungen der Files in der Working Copy

git status

geänderte Dateien im Arbeitsverzeichnis anzeigen, die für den nächsten Commit bereitgestellt werden

git add .

alle Dateien, wie sie jetzt aussiehen, zu den nächsten Commit hinzufügen (stage)

git reset [file]

eine Datei aufheben und die Änderungen im Arbeitsverzeichnis beibehalten

git diff

Änderungen zwischen Arbeitsverzeichnis und Staging-Bereich anzeigen

git commit -m “[Kommentar]”

zur Staging Area hinzugefügte Änderungen committen

BRANCH & MERGE

Isolierung der Arbeit in Zweigen, Änderung des Kontexts und Integration von Änderungen

git branch

Ihre Zweigstellen auflisten. Neben der derzeit aktiven Zweigstelle wird ein * angezeigt

git branch [branch-Name]

neuen lokalen Branch erstellen

git checkout [branch-Name]

wechseln zu einem anderem Zweig

git merge [branch-Name]

den Verlauf des angegebenen Zweigs mit dem aktuellen Verlauf zusammenführen

git branch -d [branch-Name]

lokalen Branch löschen

PRÜFEN & VERGLEICHEN

Prüfung von Protokollen, Diffs und Objektinformationen

git log

alle Commits in der Branch chronologisch anzeigen

git log branchB..branchA

die Commits auf ZweigA anzeigen, die nicht auf ZweigB sind

git log --oneline

zeigt die commits in einer Zeile an

git log --all --oneline --graph

zeigt alle commits in einer Zeile grafisch an

UPDATE & PUBLISH

Abrufen von Updates aus einem anderen Repository und Aktualisieren lokaler Repos

git push

lokale Änderungen pushen

git pull

alle Commits aus dem entfernten Verzweigungszweig abrufen und zusammenführen

git remote -v

alle konfigurierten remote Repos listen

PFADÄNDERUNGEN

Versionierung von Dateientfernungen und Pfadänderungen

git rm [file]

Löschen der Datei aus dem Projekt und Bereitstellen der Entfernung für den Commit

git mv [aktueller-path] [neuer-path]

einen bestehenden Dateipfad ändern und die Verschiebung durchführen

git log --stat -m

alle Übergabeprotokolle mit Angabe aller Pfade, die sich verschoben haben, anzeigen
